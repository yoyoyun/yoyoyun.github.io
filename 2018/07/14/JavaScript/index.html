一、如何写好JavaScript

问题1：DOM操作

  • JavaScript不要用来写改变DOM元素的样式，而应该注重状态
  • 好一点的是把样式写在css中，用js更改类名
  • 最好的是在HTML处用input+label，css用input：checked+label写样式，JavaScript处理一些后续的监听事件
  • 框架都不允许直接操作DOM，防止写坏代码；但类库如jquery为操作DOM提供了方便，无形中鼓励了写坏的代码。所以最好都从原生js写起



问题2：API的设计



  • 版本1：过程耦合+callback hell

  • 版本2：好一点的是避免过程耦合，避免callback hell，用数据抽象即可解决，把数据和状态抽取出来。但依赖于外部变量，封装性不好

  • 版本3：所以更好的是区分哪些变量应该封装，哪些应该暴露。内在使用的封装，外在关心的暴露。但可复用性比较差

  • 最好的是版本4，过程抽象，poll（做循环)，bind，高阶函数
  • 改需求:
  • 版本5: 把时间抽象出来，用Promise解决问题，抽象出wait方法，还不错
  • 版本6：高级前端，非常灵活通用，



JavaScript效率

  • 面试题
  • 版本1：复杂度 n平方
  • 版本2：赞

总结

  • JavaScript的职责
  • api的设计，通用性，抽象性，可扩展性
  • 好算法，高效率

iterator遍历器

  • 遍历器（Iterator）就是这样一种机制。它是一种接口，为各种不同的数据结构提供统一的访问机制。任何数据结构只要部署 Iterator 接口，就可以完成遍历操作（即依次处理该数据结构的所有成员）。
  • 数组（Array）、对象（Object）、Map和Set。
  • Iterator 的作用有三个：一是为各种数据结构，提供一个统一的、简便的访问接口；二是使得数据结构的成员能够按某种次序排列；三是 ES6 创造了一种新的遍历命令for…of循环，Iterator 接口主要供for…of消费。
  • 一种数据结构只要部署了 Iterator 接口，我们就称这种数据结构是“可遍历的”（iterable）。
  • ES6 规定，默认的 Iterator 接口部署在数据结构的Symbol.iterator属性，或者说，一个数据结构只要具有Symbol.iterator属性，就可以认为是“可遍历的”（iterable）。
  • 原生具备 Iterator 接口的数据结构如下。
    Array
    Map
    Set
    String
    TypedArray类似数组的对象(函数的 arguments 对象、NodeList 对象)
    Generator对象
  • 对于原生部署 Iterator 接口的数据结构，不用自己写遍历器生成函数，for…of循环会自动遍历它们。除此之外，其他数据结构（主要是对象）的 Iterator 接口，都需要自己在Symbol.iterator属性上面部署，这样才会被for…of循环遍历。

    与其他遍历语法的比较

  • 以数组为例，JavaScript 提供多种遍历语法。最原始的写法就是for循环。

这种写法比较麻烦，因此数组提供内置的forEach方法。

这种写法的问题在于，无法中途跳出forEach循环，break命令或return命令都不能奏效。

for…in循环可以遍历数组的键名。

for…in循环有几个缺点。

  • 数组的键名是数字，但是for…in循环是以字符串作为键名“0”、“1”、“2”等等。
  • for…in循环不仅遍历数字键名，还会遍历手动添加的其他键，甚至包括原型链上的键。
    某些情况下，
  • for…in循环会以任意顺序遍历键名。
    总之，for…in循环主要是为遍历对象而设计的，不适用于遍历数组。

for…of循环相比上面几种做法，有一些显著的优点。

  • 有着同for…in一样的简洁语法，但是没有for…in那些缺点。
  • 不同于forEach方法，它可以与break、continue和return配合使用。
  • 提供了遍历所有数据结构的统一操作接口。

作用域

变量屏蔽

  • JavaScript的函数在查找变量时从自身函数定义开始，从“内”向“外”查找。如果内部函数定义了与外部函数重名的变量，则内部函数的变量将“屏蔽”外部函数的变量。

变量提升

  • JavaScript的函数定义有个特点，它会先扫描整个函数体的语句，把所有申明的变量“提升”到函数顶部

全局作用域

  • 不在任何函数内定义的变量就具有全局作用域。实际上，JavaScript默认有一个全局对象window，全局作用域的变量实际上被绑定到window的一个属性
  • 全局变量与window.全局变量等价，全局函数名与window.全局函数名等价

名字空间

  • 全局变量会绑定到window上，不同的JavaScript文件如果使用了相同的全局变量，或者定义了相同名字的顶层函数，都会造成命名冲突，并且很难被发现。减少冲突的一个方法是把自己的所有变量和函数全部绑定到一个全局变量中

把自己的代码全部放入唯一的名字空间MYAPP中，会大大减少全局变量冲突的可能。
许多著名的JavaScript库都是这么干的：jQuery，YUI，underscore等等。
