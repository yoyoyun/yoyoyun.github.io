HTML简介

Hyper Text Markup Language

使用标签来描述页面的内容和结构

HTML产生

  • 1989年，Tim Berners-Lee
  • 共享文档需要
  • 还发明了浏览器、服务器和HTTP
  • HTML 1.0,1991
  • HTML 2.0,1994,IETF组织，把HTML做了一个标准化
  • HTML 3.2,1997,W3C组织
    • Netscape引入私有标签，不在HTML标准里，只有Netscape浏览器支持
    • HTML3.0失败
    • HTML之父非常不满，单干，成立W3C接管HTML标准化
  • HTML 4.01,1998
    • 样式与内容分离，CSS支持
    • Doctype
      • 指定HTML页面使用的标准和版本
      • 浏览器根据doctype决定使用哪种渲染模式
        • Quirks Mode 怪异模式
        • Almost Standard Mode 准标准模式
        • Standard Mode 标准模式
      • HTML5: <!DOCTYPE html>
  • XHTML 1.0,2000
    • 用XML语法重新定义HTML，支持扩展
    • 语法严格要求
  • XHTML 2.0
    • 不兼容历史
    • 去除样式类标签
    • 去除img、a
    • 彻底修改Form
    • 开发者不欢迎，浏览器不支持
  • HTML 5
    • 2004年，另一条线 WHATWG继续发展HTML
    • 2008年，W3C HTML5草案发布
    • 设计思想
      • 兼容已有内容
      • 避免不必要的复杂性
      • 解决现实问题
      • 优雅降级
      • 尊重事实标准
      • 用户》开发者》浏览器厂商》标准制定者》理论完美
    • HTML 5中的变化
      • doctype、meta
      • 新增语义化标签和属性 header、footer
      • 去掉纯展示性标签
      • canvas、video、audio、离线、本地存储、拖拽等
      • 语法
        • 标签不区分大小写，推荐小写
        • 空标签可以不闭合，比如input、meta
        • 属性不必引号，推荐双引号
        • 某些属性值可以省略，比如required、readonly

链接


  • a标签
    • href
      • url
        • 省略协议，根据当前页面协议解析
        • 省略协议和host，根据当前页面的协议和host自动补齐为完整的url
        • 最好用省略协议和host的绝对路径，即最开头有个/
        • 都省略，只留hash（#test），锚点，会跳转到id=“test”的位置。如果只有一个#，则跳转到顶部
    • target
      • _self，当前窗口
      • _blank，新窗口
      • 自定义，重复使用同一个tab
  • 图片img标签
    • src，图片路径，常用的是jpg和png
      • jpg
        • 照片
      • png
        • 色彩较少时使用
        • png24可以半透明，阴影效果
      • gif
        • 无法半透明
        • 可以多帧做动画
      • webp
    • alt，图片描述，盲人和省流量模式
    • width，height，建议写，以免加载完页面跳动闪烁
      • 不指定高宽：原图大小显示
      • 指定宽度：按比例缩放到指定宽度
      • 指定高度：按比例缩放到指定高度
      • 指定高宽：强制按指定宽高显示，最好跟原图一致
    • 图片的加载不会阻塞页面渲染的过程
    • 如果要添加图片说明，则用figure包起来，figcaption写图片说明

表单

使用form获取用户提供数据

  • action，路径
  • method，GET vs. POST
    • GET从服务器取数据，不会改动服务器上的内容；POST向服务器提交数据，会对服务器上的内容造成改动
    • GET请求在中间各个环节都有可能是缓存的；POST请求不会缓存
    • GET会将数据放在URL中（URL编码）发送到服务器，HTTP body内容为空，一次即可将数据发送到服务器；POST不会将数据放在路径中，而是编码后放在HTTP body中提交到服务器，分两次，先发head再发body
    • GET和POST均没有做加密

      2016.12，新的URL标准中，空格会默认替换为“+”。
    • HEAD: 只把HTTP header返回
    • PUT: 向服务器上传送文件，放置资源
    • DELETE: 删除服务器上的资源
    • OPTIONS: 没有具体的body返回，返回HTTP有哪些header选项，用于判断服务器端是否允许跨域
  • novalidate，不想使用原生HTML5的验证，用JavaScript实现验证

    表单内的元素

  • input，单行文本框
    • type
      • password，输入的文字是小圆点，移动浏览器是显示最后一个字母
      • text
      • search，在可用性上做一些辅助，读屏软件，手机键盘随之改变
      • email，在可用性上做一些辅助，@字符默认展示+输入验证
      • url，在可用性上做一些辅助，http://默认展示+输入验证
      • radio，为单选框，name为相同的值，只可以选一个
      • checkbox，为复选框，name为相同的值，可以选多个，提交为fruit=apple&fruit=banana
      • hidden，隐藏提交
      • file，上传文件，需要给表单指定enctype=“multipart/form-data”，会把表单中需要提交的字段用boundary分隔符分开几段，服务器拿到之后先分割
        • 还可同时选择多个文件，在input加上属性multiple
        • 可以指定上传文件的类型，在input加上属性accept=“image/*”
        • 有时候可以不使用原生的HTML5上传文件，可以只选择提交一个file，而不提交整个表单，可以用ajax来做
      • date，日期控件
      • datetime-local，日期+时间
      • month，月份
      • week，周
      • time，时间
      • number，上下箭头，增加减小，可以自己输入预期值
      • range，滑动条，拖拽选择数字（不显示数字，得自行用js计算），探索性，没有预期值的场景，原生的样式丑，有现成的好看的CSS组件
      • color，让用户选择一个颜色，值为#FFFFFF
    • name，向服务器发送数据的key，必填
    • value，默认值
    • placeholder，提示用户应该输入哪些内容，灰色显示
    • autofocus，自动聚焦，先把焦点放在想要用户输入的地方
    • required，必填
    • minlength，maxlength
    • pattern，匹配规则
    • 提示是浏览器自带的提示，有些情况下不友好，可借助js更改提示
    • 原生的HTML5验证，但是存在不即时的问题，在大部分的应用中，想要及时快速的将结果返回给用户，所以会借助js
    • multiple：与type=file配合使用
    • accept：与type=file配合使用
    • min，max，step：与type=number或者range配合使用
    • readonly，控件状态，会提交到服务器，但是不能改
    • disabled，控件状态，不会提交到服务器
  • textarea，多行文本框
    • cols，控制大小，几列
    • rows，控制大小，几行
  • button
    • type
      • submit，提交这个表单数据，为button的默认值
      • button，没有行为
      • reset，重置整个表单为默认状态
  • label
    • 与checkbox，radio等连用，提高可选区域，提升可用性
    • 两种方式，放在外面与input并列，用for=“id”；将input放在label里面，什么都不用
  • select，下拉框，将选项隐藏，有滚动条
    • 提交为name=1
    • multiple，可多选，选择的时候须按住shift
    • size，显示露出来几个，剩下的滚动，默认为1个
    • 选项太多可用分组，optgroup
  • output
    • 对于用户输入的输出结果
    • for，是源自哪一个用户的输入，for=input的“name”
  • 回车提交
    • 用户回车，相当于触发表单中第一个submit的button的click事件，此点需要注意。另外，不要禁用回车提交
  • 对于表单提交来说，最好用js绑定form的submit事件，而不要绑定某个button的click事件

表单设计

  • 帮助用户不出错，不要给用户太大的自由
  • 尽早提示错误
  • 扩大选择/点击区域
  • 控件较多时要分组，很长的表单要分组
  • 主要动作和次要动作，保存和取消，大多数用户会选择的操作做的明显一些

accesskey & tabindex

便于习惯用键盘操作的人

accesskey

  • 提高可用性和操作的快捷性
  • ctrl+alt+key(accesskey所定义的)
  • 用户怎么知道快捷键：通过浏览器插件，或者按下ctrl/command，会出现提示

tabindex

  • 用户习惯用键盘的tab键往前找到下一个focus的元素
  • 数字，规定按tab依次导航的顺序，如果不想导航设置为-1

id, class & style

  • id保证唯一性
  • class多用在CSS
  • style指定内联样式，尽量避免使用

contenteditable & spellcheck

与文本内容相关

contenteditable

  • 加了这个属性之后，在页面中的内容可以一边展示一边修改

spellcheck

  • 拼写检查
  • spellcheck不是布尔型的值，加这个属性的时候，必须写等于什么，spellcheck = “true”

语言lang & dir

lang

  • 多语言版本，搜索引擎

    dir

  • 书写方向，默认是从左到右；阿拉伯语从右到左 dir=“rtl” (right to left)

title

  • 完整解释/文字截断(CSS用…做了省略)后仍能看到完整解释
  • 但是体验不是特别好，有延时，所以一般用js做代替title

hidden

  • 相当于CSS的display:none(display重新显示要根据块级元素和行内元素恢复为对应的值)
  • 可以用js操纵值等于true和false
  • 读屏软件，可用性改进

无障碍性

  • 或可访问性，Accessibility
  • 确保任何人都有办法获取放在网页上的媒体内容
  • 不让身体、心理或技术上的问题成为获取信息的障碍

Web开发者应该做的事情

WCAG(Web Content Accessibility Guidelines)

  • 比较早出来，关注纯内容性的规范，图片，文本
  • 有相应的浏览器插件检查

ARIA(Accessible Rich Internet Applications)

  • 针对web富应用做的规范，下拉列表，菜单
  • 读屏软件
  • 可以在标签加上aria属性，告诉辅助性工具这个元素是什么含义，role=“tablist”，在文档中有很详细的属性

提升无障碍性

  • 为img提供alt属性
  • noscript
  • input和label对应
  • 图形验证码与语音验证码
  • 文字和背景有足够的对比度
  • 键盘可操作

语义化

  • HTML中的元素、属性及属性值都拥有某些含义
  • 开发者应该遵循语义来编写代码

为什么语义化很重要？

  • 提升代码可读性、可维护性
  • 搜索引擎优化，搜索引擎判断内容相关，词语出现的频次
  • 提升无障碍性

meta标签

与页面相关的内容

data-*

  • 写在data-后面的内容会放在dataset中
  • js可以用$0表示当前元素

microdata

  • HTML5中的一个规范
  • 在HTML中通过属性嵌入格式化数据
  • 提供给搜索引擎、浏览器（插件）使用
  • 是一本书？一个人？
  • itemscope 标签之间是一个实体
  • itemtype 实体类型是人
  • itemprop 实体属性
  • 把一段文字组织成了一段格式化的数据，方便搜索引擎显示一个饭店的评价数等
  • http://schema.org/Person是[实体类型定义网站](http://schema.org/Person)

JSON-LD

  • 可以把microdata写在一处，就是JSON-LD
