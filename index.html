#ubuntu14.04安装apache2+php+mysql
sudo apt-get update

安装Apache2：

sudo apt-get install apache2

安装PHP模块：

sudo apt-get install php

安装Mysql：

sudo apt-get install mysql-server

输入root用户密码

安装phpmyadmin

其他模块安装：

sudo apt-get install libapache2-mod-php

sudo apt-get install libapache2-mod-auth-mysql

上面两个的作用是，apache能够解析php，需要借助这两个模块来找到php engine

sudo apt-get install php-mysql

上面语句是在php操作mysql数据库时用到

sudo apt-get install php-gd

上面语句是GD库

法一：

apache2的默认目录配置在/etc/apache2/sites-enabled/00default文件中

找到DocumentRoot项，将/var/www改为目标目录即可

sudo ln -s /home/username/phptest /var/www/phptest

Mac OS彻底解决文件被os x系统占用无法打开问题

Posted on 2018-01-07

问题

Mac用mounty挂载移动硬盘，编辑过的文件，有时候会出现“xx.xx已被OS X使用，无法打开”的问题。

解决方案

打开终端，进入硬盘目录

终端 ls -al 命令一下就可以看到：-rwxr-xr-x@，文件的读写权限中被加上了一个@符号，这就是 Mac OS X 文件系统的附加属性（extended attributes）。

在终端里输入 xattr -l ，然后把出问题的文件拖动到 “xattr -l”后面,回车，可以查看详细的附加属性内容,可看到文件中多了扩展属性com.apple.FinderInfo。

查到博客解决方案推荐：

确实能够删除扩展属性，但是发现过了不久又重新出现文档变灰无法打开情况，须重复执行命令。

彻底解决方案

-c 参数去除当前文件夹中所有 txt 的附加属性。再次 ls -al 可以发现没有了@符号。

过一段时间之后也并未出现无法打开情形，可认为此问题已彻底解决。
