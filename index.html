HTML简介

HTML产生

  • 1989年，Tim Berners-Lee
  • 共享文档需要
  • 还发明了浏览器、服务器和HTTP
  • HTML 1.0,1991
  • HTML 2.0,1994,IETF组织，把HTML做了一个标准化
  • HTML 3.2,1997,W3C组织
    • Netscape引入私有标签，不在HTML标准里，只有Netscape浏览器支持
    • HTML3.0失败
    • HTML之父非常不满，单干，成立W3C接管HTML标准化
  • HTML 4.01,1998
    • 样式与内容分离，CSS支持
    • Doctype
      • 指定HTML页面使用的标准和版本
      • 浏览器根据doctype决定使用哪种渲染模式
        • Quirks Mode 怪异模式
        • Almost Standard Mode 准标准模式
        • Standard Mode 标准模式
      • HTML5: <!DOCTYPE html>
  • XHTML 1.0,2000
    • 用XML语法重新定义HTML，支持扩展
    • 语法严格要求
  • XHTML 2.0
    • 不兼容历史
    • 去除样式类标签
    • 去除img、a
    • 彻底修改Form
    • 开发者不欢迎，浏览器不支持
  • HTML 5
    • 2004年，另一条线 WHATWG继续发展HTML
    • 2008年，W3C HTML5草案发布
    • 设计思想
      • 兼容已有内容
      • 避免不必要的复杂性
      • 解决现实问题
      • 优雅降级
      • 尊重事实标准
      • 用户》开发者》浏览器厂商》标准制定者》理论完美
    • HTML 5中的变化
      • doctype、meta
      • 新增语义化标签和属性 header、footer
      • 去掉纯展示性标签
      • canvas、video、audio、离线、本地存储、拖拽等
      • 语法
        • 标签不区分大小写，推荐小写
        • 空标签可以不闭合，比如input、meta
        • 属性不必引号，推荐双引号
        • 某些属性值可以省略，比如required、readonly

使用form获取用户提供数据

  • action，路径
  • method，GET vs. POST
    • GET从服务器取数据，不会改动服务器上的内容；POST向服务器提交数据，会对服务器上的内容造成改动
    • GET请求在中间各个环节都有可能是缓存的；POST请求不会缓存
    • GET会将数据放在URL中（URL编码）发送到服务器，HTTP body内容为空，一次即可将数据发送到服务器；POST不会将数据放在路径中，而是编码后放在HTTP body中提交到服务器，分两次，先发head再发body
    • GET和POST均没有做加密

      2016.12，新的URL标准中，空格会默认替换为“+”。
    • HEAD: 只把HTTP header返回
    • PUT: 向服务器上传送文件，放置资源
    • DELETE: 删除服务器上的资源
    • OPTIONS: 没有具体的body返回，返回HTTP有哪些header选项，用于判断服务器端是否允许跨域
  • novalidate，不想使用原生HTML5的验证，用JavaScript实现验证

accesskey & tabindex

便于习惯用键盘操作的人

accesskey

  • 提高可用性和操作的快捷性
  • ctrl+alt+key(accesskey所定义的)
  • 用户怎么知道快捷键：通过浏览器插件，或者按下ctrl/command，会出现提示

tabindex

  • 用户习惯用键盘的tab键往前找到下一个focus的元素
  • 数字，规定按tab依次导航的顺序，如果不想导航设置为-1

id, class & style

  • id保证唯一性
  • class多用在CSS
  • style指定内联样式，尽量避免使用

  • spellcheck不是布尔型的值，加这个属性的时候，必须写等于什么，spellcheck = “true”

Mac OS彻底解决文件被os x系统占用无法打开问题

Veröffentlicht am 2018-01-07

问题

Mac用mounty挂载移动硬盘，编辑过的文件，有时候会出现“xx.xx已被OS X使用，无法打开”的问题。

解决方案

打开终端，进入硬盘目录

终端 ls -al 命令一下就可以看到：-rwxr-xr-x@，文件的读写权限中被加上了一个@符号，这就是 Mac OS X 文件系统的附加属性（extended attributes）。

在终端里输入 xattr -l ，然后把出问题的文件拖动到 “xattr -l”后面,回车，可以查看详细的附加属性内容,可看到文件中多了扩展属性com.apple.FinderInfo。

查到博客解决方案推荐：

确实能够删除扩展属性，但是发现过了不久又重新出现文档变灰无法打开情况，须重复执行命令。

彻底解决方案

-c 参数去除当前文件夹中所有 txt 的附加属性。再次 ls -al 可以发现没有了@符号。

过一段时间之后也并未出现无法打开情形，可认为此问题已彻底解决。
